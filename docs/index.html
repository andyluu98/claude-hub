Claude Hub

A Windows-native dashboard for managing multiple Claude Code sessions in parallel. Real PTY terminals piped to xterm.js — the actual interactive TUI, not a fake chat wrapper.

Features

Everything you need to run multiple Claude Code agents side by side

🖥

Real PTY Terminals

Each session spawns a real claude.exe process via node-pty. Raw bytes pipe through WebSocket to xterm.js — full TUI rendering, colors, prompts, everything.

📋

Kanban Lanes

Organize sessions into Active, Pending, and Done lanes. Drag sessions between lanes as your workflow progresses.

📂

Filesystem Browser

Built-in file explorer with bookmarks, recent paths, right-click context menu. Create, rename, duplicate, delete files — all from the sidebar.

📋

Persistent State

Sessions survive page reloads and server restarts. Terminal scrollback is preserved — pick up exactly where you left off.

📎

Clipboard Image Paste

Paste images from clipboard directly into the terminal. Auto-saved to a temp file and pushed into Claude's input as a file reference.

🔒

Localhost Only

Binds to 127.0.0.1, CSRF protection via Origin check. Designed for single-user local development — no auth needed.

Quick Start

1

Clone the repo

git clone https://github.com/andyluu98/claude-hub.git

2

Install dependencies

cd claude-hub && npm install

Requires Visual Studio Build Tools for node-pty native compilation.

3

Launch

start.bat or node server.js

Browser auto-opens at http://localhost:8765

4

Create sessions

Pick a working directory, click + New Session, and start coding with Claude.

Architecture

Browser (xterm.js) | | WebSocket (raw bytes) v Express Server (:8765, 127.0.0.1) | | node-pty (spawn) v claude.exe x N sessions | | PTY I/O v Your codebase
